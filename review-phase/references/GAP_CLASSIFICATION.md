# Gap Classification

## HIGH_URGENCY
Use for issues that must be fixed before acceptance or before dependent work proceeds.

Examples:
- primary business objective not fulfilled
- major plan mismatch
- broken or unsafe behavior
- serious regression risk
- critical test gap
- architecture/design violation with near-term impact

## MEDIUM_URGENCY
Use for important issues that should be fixed soon.

Examples:
- partial spec miss
- notable edge-case weakness
- incomplete validation
- maintainability problem that raises medium-term risk
- business/workflow friction that weakens the intended value

## LOW_URGENCY
Use for minor issues that do not usually block acceptance.

Examples:
- minor naming or structure cleanup
- small documentation inconsistency
- modest test improvement
- local non-critical refactor opportunity

## FUTURE_IMPROVEMENT
Use for intentionally deferred or optional work outside current acceptance scope.

Examples:
- broader optimization
- non-essential abstraction improvement
- enhancement outside current plan/roadmap
