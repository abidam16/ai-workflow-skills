# Review Dimensions Checklist

Use these dimensions in every review.

## 1. Business fulfillment
- requirement/problem solved?
- user workflow supported?
- business rules preserved?
- success direction supported?

## 2. Objective fulfillment
- primary objective achieved?
- partially achieved?
- missed?

## 3. Scope discipline
- in-scope items implemented?
- out-of-scope respected?
- unrelated changes introduced?

## 4. Detailed-spec compliance
- required behavior present?
- edge cases handled as expected?
- constraints respected?
- important assumptions preserved?

## 5. Design / architecture alignment
- fits intended design?
- follows architecture rules?
- avoids harmful coupling or pattern violations?

## 6. Code quality
- readability
- maintainability
- consistency
- cohesion
- duplication
- naming clarity

## 7. Validation and testing
- required tests present?
- tests meaningful?
- validation evidence sufficient?
- failure paths covered when needed?

## 8. Integration impact
- local integration sound?
- cross-task interactions healthy?
- future tasks blocked or complicated?
- docs need update?
