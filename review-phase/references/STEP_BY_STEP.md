# Step-by-Step Review Procedure

## 1. Establish the target
Define exactly what is under review and what mode applies.

## 2. Gather source artifacts
List the source artifacts used for judgment.

## 3. Read intent first
Read plan / roadmap / PRD / ADR before reading the code diff or changed files.

## 4. Assess business fulfillment
Check the implementation against the intended business need, user workflow, rules, and success direction.

## 5. Assess plan / roadmap / design fulfillment
Check objective, scope, detailed spec, constraints, and acceptance direction.

## 6. Assess technical quality
Check maintainability, consistency, architectural fit, readability, duplication, and risk.

## 7. Assess validation and tests
Check whether required tests exist, whether they are meaningful, and whether validation evidence is sufficient.

## 8. Assess integration impact
Check local and cross-task integration, future-task impact, and hidden blockers.

## 9. Classify findings
Assign `HIGH_URGENCY`, `MEDIUM_URGENCY`, `LOW_URGENCY`, or `FUTURE_IMPROVEMENT`.

## 10. Decide status
Choose one status: `APPROVED`, `APPROVED_WITH_MINOR_IMPROVEMENTS`, `NEEDS_REVISION`, or `BLOCKED`.

## 11. Write the report
Use the appropriate template and make next actions explicit.
