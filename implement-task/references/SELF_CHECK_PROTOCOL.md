# SELF_CHECK_PROTOCOL

Before claiming completion, compare the implementation against each relevant plan section.

## Required checks

### Objective
- fulfilled / partially fulfilled / not fulfilled

### Scope
- in-scope implemented?
- out-of-scope respected?

### Detailed specification
- all required behaviors covered?
- all required constraints preserved?

### Files / components
- expected files changed?
- prohibited areas avoided?

### Validation and tests
- required tests added/updated/run?
- validation evidence sufficient?

### Review expectations
- obvious review failures already addressed?

## Final self-check output

State clearly:
- what is fulfilled
- what is partial
- what is missing
- any deviation
- any blocker still remaining
