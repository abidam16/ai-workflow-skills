# REVIEW_GAP_CLASSIFICATION

Use these urgency levels when reporting remaining gaps after implementation.

## High urgency
Must be fixed before acceptance.
Examples:
- objective not met
- major missing scope item
- dangerous deviation
- core behavior broken
- critical missing test

## Medium urgency
Should be fixed soon and may block acceptance depending on context.
Examples:
- partial spec miss
- important edge case not covered
- weak validation evidence
- maintainability issue with near-term impact

## Low urgency
Does not usually block acceptance, but should be tracked.
Examples:
- small cleanup
- naming inconsistency
- minor documentation update
- limited test improvement opportunity
