# ADR Structure Guide

Use this structure:

1. **Title**
   - Name the decision directly.

2. **Status**
   - Proposed / Accepted / Rejected / Superseded

3. **Context and Problem Statement**
   - What problem or pressure required this decision?

4. **Decision Drivers**
   - What criteria matter most?

5. **Considered Options**
   - List the serious alternatives.

6. **Decision Outcome**
   - State the chosen option and why.

7. **Consequences**
   - Positive
   - Negative
   - Follow-up / neutral impact

8. **Scope and Impact**
   - Which systems, modules, teams, or workflows are affected?

9. **Non-Goals / Not Addressed**
   - What this ADR does not decide.

10. **Related Artifacts**
   - PRD, roadmap, plans, prior ADRs, implementation constraints

Keep the ADR small and decision-oriented. Avoid turning it into a broad design specification.
