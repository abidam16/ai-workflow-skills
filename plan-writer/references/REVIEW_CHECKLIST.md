# Review Checklist

A finished implementation should be reviewed against the plan using at least these checks:

- objective fulfilled
- in-scope items implemented
- out-of-scope respected
- detailed specification followed
- expected files/components changed appropriately
- forbidden or unrelated areas not touched without justification
- validation requirements satisfied
- required tests added, updated, or run
- no silent divergence from the plan
- trade-offs and risks remain acceptable
- future improvements were not incorrectly pulled into current scope
